Libellé consigne

Consigne générique ATLAS :

 

1 – Consultation de la log

Ouvrir la log du job en ABORT, consulter son contenu puis copier le message d erreur dans le ticket Incident. Y ajouter également la date de l’ABORT, le serveur concerné, le job, le jobset et le qualifier.

 

Les logs se trouvent à cet emplacement :

/apps/atlas/atlas2v0/uf1/data1/imp

 

Il faut faire une recherche sur le nom du job (ou du jobset) :

ls -ltr |grep -i <nom_job>

 

 

2 – Détermination de l’environnement concerné : Atlas, Orion

Pour connaitre l emplacement du traitement :

- se connecter à l'environnement avec le user applicatif en tapant la commande su - atlas

- type <nom_du_script>   => permet de connaître le chemin du script

Si cette commande ne donne rien, chercher le script dans /apps/atlas/atlas2v0/uf1/dwhexeunl0XX/jcl où XX correspond à la version infocentre (prendre le nombre le plus grand)

 

Si le script est dans : /apps/orion/<version>/uf1 => job ORION

Si le script est dans : /apps/atlas/atlas2v0/uf1 => job ATLAS

Si le script est dans : /apps/atlas/atlas2v0/uf1/dwhexeunl0XX/jcl => job infocentre

 

3 – Détermination du type de traitement : standard, spécifique, autre

Les jobs se trouvent dans differents sous-repertoires (en fonction de leur type).

Ces sous-répertoires se trouvent a la racine de l environnement.

Si Environnement Atlas => racine_env  = /apps/atlas/atlas2v0/uf1

Si Environnement Orion => racine_env = /apps/orion/<version>/uf1

 

4 – Actions à mener

Si le traitement est dans le répertoire :

1)     <racine_env>/jcl

C’est un job standard donc il faut assigner le ticket aux etudes puis : 

-          En HO : leur envoyer un mail

-          En HNO : appeler la maintenance ATLAS d’astreinte

 

2)     <racine_env>/site/jcl

C’est un job specifique donc :

-          En HO : envoyer un mail à BDSI OCEAN, en  copie les GPMX avec le numero d incident correspondant

-          En HNO : appeler l astreinte BDSI OCEAN

 

3)     <racine_env>/jclsite

Si le traitement fait partie des jobsets suivants :

j9ar

j9f1 j9b1

j9f2 j9b2

j9r1

 

Affecter le ticket a la MEO IRB:

-          En HO : cette action suffit

-          En HNO : appeler l astreinte MEO IRB

 

Sinon c’est pour le metier BDSI OCEAN :

-          En HO : envoyer un mail au metier copie les GPMX avec le numero d incident correspondant

-          En HNO : appeler l astreinte BDSI OCEAN 

 

4)     /apps/atlas/atlas2v0/uf1/dwhexeunl0XX/jcl

C’est un job infocentre standard donc il faut assigner le ticket aux etudes puis 

-          En HO : leur envoyer un mail

-          En HNO : appeler astreinte maintenance INFOCENTRE

 

En cas de doute, passer par la MEO IRB.
